**Segmented control** (custom, NOT a shadcn component):
- 3 segments: "Open", "Resolved", "All" per CONTEXT.md (NOT 4 segments)
- Default selected: "Open" via `useState<'open' | 'resolved' | 'all'>('open')`
- Container styling: `inline-flex rounded-lg bg-gray-100 dark:bg-gray-800 p-1`
- Active segment: `bg-white dark:bg-gray-700 text-gray-900 dark:text-white [box-shadow:var(--card-shadow)] rounded-md px-3 py-1.5 text-sm font-medium transition-all`
- Inactive segment: `text-gray-500 dark:text-gray-400 hover:text-gray-700 dark:hover:text-gray-300 rounded-md px-3 py-1.5 text-sm font-medium transition-all`
- This MUST be visually distinct from TabNav's underline style -- it's a pill/toggle control
- Claude's discretion: optionally show counts per segment like "Open (3)"

**Filter logic:**
- "Open" filter shows alerts where `status === 'open'`
- "Resolved" filter shows alerts where `status === 'resolved'`
- "All" shows all alerts
- Filter is applied via `useMemo` or inline `.filter()` on the alerts array

**Table structure** (use shadcn Table components):
- Wrapped in the standard card anatomy div: `rounded-xl border border-gray-200 dark:border-gray-800 bg-white dark:bg-[var(--card-bg)] [box-shadow:var(--card-shadow)] p-5`
- SectionHeader with title "Issues" and a ghost RefreshCw icon button as the action prop
- Segmented control below SectionHeader, above table, with `mt-3 mb-4` spacing
- Table columns: Issue | Worker | Severity | Status | Date
- Table header cells: `text-xs font-medium text-gray-500 dark:text-gray-400 uppercase tracking-wide`

**Severity badges** using shadcn Badge:
- Do NOT pass a `variant` prop (let it be undefined or pass `variant="secondary"`). Instead, rely entirely on `className` for colors to avoid conflicts with default bg-primary:
  ```typescript
  const SEVERITY_STYLES: Record<Severity, string> = {
    high: 'bg-red-100 text-red-700 dark:bg-red-950 dark:text-red-400 border-transparent',
    medium: 'bg-amber-100 text-amber-700 dark:bg-amber-950 dark:text-amber-400 border-transparent',
    low: 'bg-gray-100 text-gray-600 dark:bg-gray-800 dark:text-gray-400 border-transparent',
  }
  ```
- Capitalize severity text: "High", "Medium", "Low"

**Status dots:**
```typescript
const STATUS_DOT_COLORS: Record<AlertStatus, string> = {
  open: 'bg-amber-400',
  resolved: 'bg-emerald-400',
}
```
- Render as: `<span className="inline-block h-2 w-2 rounded-full {color}" />` + capitalized status text
- Wrap in a flex container with `gap-2 items-center`

**Row styling:**
- Primary text (Issue): `text-sm font-medium text-gray-900 dark:text-white`
- Secondary text (Worker, Date): `text-sm text-gray-500 dark:text-gray-400`
- Row hover: `hover:bg-gray-50 dark:hover:bg-gray-800/50`
- No click behavior (v2-01 scope)

**Date formatting** (Claude's discretion): Use relative format for today ("2 hours ago"), absolute "Mar DD" for older dates. A simple helper function is fine -- no need for a date library.

**Empty state:**
- When `alerts.length === 0`, render EmptyState inside the card (below the SectionHeader) with:
  - icon: ShieldAlert
  - title: "No issues reported"
  - description: "Safety alerts and issues reported by workers will appear here."
- Hide the segmented control when no alerts (nothing to filter)

Follow established patterns: `'use client'`, named export, named Props interface.
